# Writing Notes

- **dual process theory**
    - group of theories that share the common idea of two types of cognitive processing (*Evans & Stanovich (2013)*)
        - **type 1:** intuitive, autonomous processing
            - also called "system 1" (*Kahnemann (2011)*)
        - **type 2:** reflective, decoupled processing
            - also called "system 2" (*Kahnemann (2011)*)
    - different theories prescribe different attributes to the different types of processing beyond the descriptions above (*Evans & Stanovich (2013)*)
        - more consistency with definition of type 2 processing
        - differences in theories for type 1 processing
        - table below is mostly consistent among authors

        ```
        | Type 1           | Type 2              |
        | ---------------- | ------------------- |
        | fast             | slow                |
        | parallel         | serial              |
        | nonconscious     | conscious           |
        | biased responses | normative responses |
        | automatic        | controlled          |
        ```

    - theories also differ in the relation of the two systems (*Evans & Stanovich (2013)*)
        - **parallel-competitive** relation, asserting both systems work in parallel and resolve conflicts if required
        - **default-interventionist** relation, assuming type 1 processing yields a default response, that type 2 processing may intervene with (*this thesis is assuming this basically*)
        - concerns raised about parallel-competitive hypothesis, since type 2 reasoning is slower than type 1 reasoning, which means, type 1 reasoning would need for type 2 reasoning to finish
- **heuristics**
    - method for problem solving, that simplifies the problem in favour of a quicker solution
        - "A heuristic is a strategy that ignores part of the information, with the goal of making decisions more quickly, frugally, and/or accurately than more complex methods." (*Gigerenzer & Gaissmaier (2011)*)
    - framing for this thesis: heuristics should be presented neutrally, highlighting both benefits (efficiency, speed, frugality) and shortcomings (potential biases, errors in certain contexts)
        - intended takeaway: heuristics are generally useful and efficient, but their limitations should be made explicit
    - theory of bounded rationality (*Simon (1956)*)
        - decisions are made with limited
            - time
            - mental resources
            - information
        - results in a phenomenon called **satisficing**
            - term coined by *Simon (1956)* as portmanteau of "satisfy" and "suffice" (*Hjeij and Wilks (2023)*)
            - "Rather than trying to find the ideal option, people choose the first acceptable or satisfactory option they find" (*Hjeij and Wilks (2023)*)
    - computer science likely has the biggest share of deliberately applied heuristics (*Hjeij and Wilks (2023)*)
        - classic problems like travelling salesman problem or chess for heuristic algorithms
            - find acceptable solutions without exploring full solution space
            - use heuristics to narrow down solution space to promising candidates
        - machine learning (and "AI"), as well as NLP apply heuristics
